Monasca

Monitoring-as-a-Service (at-Scale)

deployment

Monasca is a system built for large scale deployments. One side effect of this is there are many components in the system each of which is an installation in and of itself. The end result of this is that I highly recommend you use a configuration management system to install Monasca.

The HP Monasca team has published a set of Ansible roles that can be used to install and configure Monasca on Debian/Ubuntu based machines. These roles all are in repos that start with ‘ansible-’ and are available on github under the hpcloud-mon organization. These roles are used in the Monasca Vagrant development environment but are also used internally at HP for full HA installations of Monasca.

In addition to the Ansible roles a Puppet module is maintained by the Monasca community in the puppet-monasca git repository.
